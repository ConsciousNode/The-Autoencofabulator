Best Practices Guide
Corpus
· Use 10+ files per format. More homogeneous = faster convergence.
· Tag files meaningfully — tags become the generation vocabulary.
· BMP: minimum 8×8px, 24-bit. WAV: mono preferred for v0.2.
Training milestones (TXT/HTML)
· ~30k steps — loss trending down, output still noise
· ~200k steps — occasional character clusters
· ~500k steps — word-like fragments, tag echoes
· ~1M+ steps — recognizable structure
Loss interpretation
· Jumpy loss is normal — timestep is sampled randomly each step.
· Watch the rolling average (shaded region), not individual spikes.
· If average not decreasing after 50k steps → try higher LR (5e-4).
Generation
· Try Deterministic (DDIM) if output is very noisy.
· More steps (200) = cleaner output, slower generation.
· Export .fabula before closing the browser!
